# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## このリポジトリは何か

`ward` プロジェクトの **実装本体**。AIエージェント（Claude）に**本物の自宅インフラ（NUC）を運用させ**、「どこまで自律を許せるか × 事故をどう封じ込めるか（ガードレール）」を研究する。ここで育てるのは IaC・ガードレールのポリシー・評価スクリプトなど「動くもの」。

- **憲章・リサーチクエスチョン・ロードマップ・作業ログ** → 別リポジトリ `ward-doc`（ドキュメント専用）。判断に迷ったら `ward-doc/CONCEPT.md` をまず読む。ローカルでは兄弟ディレクトリ `../ward-doc` にクローン済み。
- このリポジトリ（`ward`）に足すのは**実装**。新しい種類のものを足すときは、まず `ward-doc/CONCEPT.md` の方針と下の原則に沿うか確認する。

## 守るべき原則（実作業の判断基準）

1. **Single Source of Truth** — インフラの状態はこのリポジトリのコードで表す。手作業の変更を "正" にしない。
2. **すべては差分で残る** — 変更は必ず diff（コミット/PR）として記録 ＝ レビュー可能・巻き戻せる。
3. **再現性ファースト** — 土台はいつでもコードから作り直せること。"出どころ不明" な既存状態は信用せず、コード化 or 作り直す。
4. **段階的に自律を解放する** — まず読み取り、次に承認付き、最後に自律。一気にやらない。

## NUC（research substrate）の触り方

- host は自宅の **NUC7i5BNH / Ubuntu 24.04 / 2C4T・16GB・232GB SSD 1枚**（単一ディスク＝冗長性なし）。スペック前提でスケールを考える（重い処理を前提にしない）。
- Mac から **Tailscale SSH** でアクセス。ユーザ `ruribou` は `sudo` と `docker` グループ所属。
- **破壊的変更は、その変更を "コード" として用意できるまで行わない。** まず read-only で実態を確認してから動く（過去に「まっさらのつもり」が実際は k3s 稼働中だった前例あり）。
- **`sudo` はパスワードを要求し、非対話の `sudo -n` はブロックされる。** 特権が要るコマンドは、自分で実行せず**ユーザにセッション内で `! sudo …` として打ってもらう**。
- `docker` はグループ所属のため非 sudo で叩ける。
- **このリポジトリは public。** Tailscale IP・ホスト固有のアドレス・鍵類を、ファイル・コミット・PR・ログのどこにも書かない（`.gitignore` で secrets を厳しめに除外済み）。実 IP/ホスト名が要る値は inventory 等の "テンプレート＋gitignore された実体" に分離する。

## Git / GitHub の進め方

- リモート `git@github.com:ruribou/ward.git`（**public**、デフォルトブランチ `main`）。`gh` は `ruribou` で認証済み。
- **直 push しない。main へは PR 経由**（原則#2の実践）。ブランチを `main` から切り → push → `gh pr create --base main`。（※リポジトリ立ち上げの初期 scaffold コミットのみ、空リポジトリへ PR できないためブートストラップ例外として main 直コミット済み。）
- PR/ログでは「**NUC本体に変更を加えたか / ドキュメントのみか**」を明示する（事故の追跡可能性のため）。

## 進め方のトーン

- ユーザは**ゆっくり・学び重視**で進めたい。1 歩ずつ、**なぜそうするかを説明しながら**。一気に全部組まない。
- 大きな技術選択（substrate・ツール）は勝手に確定せず、選択肢と傾きを出して合意を取る。
