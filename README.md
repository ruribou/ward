# ward

> **W**atched **A**utonomy, **R**eversible **D**eployment —
> AIエージェント（Claude）に**本物の自宅インフラ**を運用させ、
> 「どこまで自律を許せるか × 事故をどう封じ込めるか（ガードレール）」を研究するプロジェクト。

このリポジトリ `ward` は ward の **実装本体**。IaC・ガードレールのポリシー・評価スクリプトなど、
「動くもの」をここで育てる。**憲章・ロードマップ・作業ログ・事故譚**は別リポジトリ
[`ward-doc`](https://github.com/ruribou/ward-doc) にある。

## 名前について

`ward` という語がこのプロジェクトを二重に言い当てている：

- **ward（動詞）＝受け流す・防ぐ** … 事故を *ward off* する。ガードレールで blast radius を封じ込める側面。
- **ward（名詞）＝後見下にある被後見人** … AIエージェントを最初は強い監督下に置き、信頼に応じて
  **段階的に自律を解放していく "ward"** として扱う側面。

頭文字を拾うと **W**atched **A**utonomy, **R**eversible **D**eployment（監督付きの自律 × 巻き戻せる変更）。

## 原則

1. **Single Source of Truth** — インフラの状態は、このリポジトリのコードで表す。
2. **すべては差分で残る** — AI/人間の変更は必ず diff（コミット/PR）として記録される＝巻き戻せる。
3. **再現性ファースト** — 土台はいつでもコードから作り直せる。"出どころ不明" な状態は信用しない。
4. **段階的に自律を解放する** — まず読み取り、次に承認付き、最後に自律。一気にやらない。

## いま
