# ward

> 自宅インフラを、自然言語で・安全に、Claude Code から操作する。

_English: [README.md](./README.md)_

**ward** は、AI（[Claude Code](https://claude.com/claude-code) 上の Claude）に自宅インフラの操作を
[MCP](https://modelcontextprotocol.io) ツールとして行わせる仕組みです。ディスク確認、コンテナ一覧、
イメージの pull — そういう操作を任せます。

難しいのは AI とマシンを繋ぐことじゃありません。**どこまで自律を許すか、事故をどう封じ込めるか**です。
ward はそこを中心に作っています。どの操作も、何かに触れる前に必ずガードレールを通る。自律は一気に
じゃなく一段ずつ — まず読み取りだけ、次に承認付き、完全な自律は最後です。

## 名前について — なぜ _ward_ か

この語は、プロジェクトの性格を二重に言い当てています。

- **ward（動詞）＝受け流す・防ぐ。** 事故を _ward off_ する。ガードレールで blast radius を封じ込める側面。
- **ward（名詞）＝後見下にある被後見人。** AI を最初は強い監督下に置いて、信頼に応じて少しずつ自律を
  解放していく "ward" として扱う側面。

頭文字を拾えば **W**atched **A**utonomy, **R**eversible **D**eployment（監督付きの自律 × 巻き戻せる
変更）。プロジェクトの2本柱そのものです。

## 安全モデル

ward は既定で読み取り専用。エージェントが行儀よくしてくれる前提は置きません。効くのは信頼じゃなく、
構造です。

- **単一の真実源。** ward が実行できる操作は、全部 [`operations.yaml`](./operations.yaml) に固定の
  コマンドと `risk`（リスク）として書いてあります。この許可リストの外には何も届きません。
- **ガードレールゲート。** どの呼び出しも、実行の前に `autonomy（自律レベル） × risk` で判定します。

  | autonomy \ risk | read-only | mutating   |
  | --------------- | --------- | ---------- |
  | `read-only`     | 許可      | 禁止       |
  | `approval`      | 許可      | 承認が必要 |

- **読み取り専用のフロア。** 既定レベルは `read-only`。ここでは mutating な操作はツールとして姿すら
  見せません。実行できない書き込みを、エージェントはそもそも目にしないわけです。
- **書き込みは propose → approve。** `approval` レベルでも、mutating な操作は頼まれただけじゃ動きません。
  いったん*提案として stage* して、別の承認ステップを踏んで初めて実行します。モデルが一手で書き込む
  ことはできず、人間が必ず間に入る作りです。
- **定数コマンド・no-shell。** コマンドは固定の引数列で実行して、シェルは通しません。しかもモデルは
  コマンドを一切組み立てません（引数ひとつ渡さない）。注入する隙がそもそもないんです。
- **追記専用の監査ログ。** 何を提案して何を実行したかを残します。どの操作も、後から見直せて・巻き
  戻せる状態に保ちます。

## 自律ラダー

ward は、エージェントに許す範囲を一段ずつ広げていきます。段を上げるのは設定の変更であって、作り直し
じゃありません。

1. **`read-only`** _(既定)_ — 見るだけ。基盤には手を出せない。
2. **`approval`** — 変更を*提案*できる。提案はひとつずつ人間が承認してから実行する。
3. **`autonomous`** — ポリシーの範囲内なら、操作ごとの承認なしで動く。**未実装。**

## 現在地

まだ初期段階で、育てている最中です。ここまでの歩み:

- **M1** — MCP 経由の読み取り専用ステータス
- **M2** — 操作を YAML レジストリで宣言
- **M3** — 承認ゲートと、初の mutating な操作
- **i18n** — ツールの文言とメッセージの二言語化（既定は英語、`WARD_LANG=ja` で日本語）

**インストールとクイックスタートのドキュメントはこれから** — インターフェースが落ち着き次第そろえます。
それまでの計画は [issues](https://github.com/ruribou/ward/issues) にあります。
