# ward

> 自宅インフラを、自然言語で・安全に、Claude Code 経由で操作する。

_English: [README.md](./README.md)_

**ward** は、AIエージェント（Claude、[Claude Code](https://claude.com/claude-code) 経由）に、
自宅インフラへの実際の操作 — ディスク確認・コンテナ一覧・イメージの pull など — を
[MCP](https://modelcontextprotocol.io) ツールとして実行させる仕組みです。

難しいのは AI とマシンを繋ぐことではありません。**どこまで自律を許すか、そして事故が
起きたときどう封じ込めるか**です。ward はこの問いを中心に設計されています。すべての
操作は、何かに触れる前に必ずガードレールを通過する。そして自律は一段ずつ意図的に
解放する — まず読み取り、次に人間の承認付き、完全な自律は最後に。

## 名前について — なぜ _ward_ か

この語は、プロジェクトを二重の意味で言い当てています：

- **ward（動詞）＝受け流す・防ぐ。** 事故を _ward off_ する＝ガードレールで blast radius
  を封じ込める側面。
- **ward（名詞）＝後見下にある被後見人。** AIエージェントを最初は強い監督下に置き、
  信頼に応じて段階的に自律を解放していく "ward" として扱う側面。

頭文字を拾うと **W**atched **A**utonomy, **R**eversible **D**eployment（監督付きの自律 ×
巻き戻せる変更）— プロジェクトの2本柱そのもの。

## 安全モデル

ward は既定で読み取り専用であり、エージェントの行儀の良さを前提にしません。保証は
信頼ではなく構造から来ます：

- **単一の真実源。** ward が実行できる操作はすべて
  [`operations.yaml`](./operations.yaml) に、固定コマンドと `risk`（リスク）として
  宣言されています。この許可リストの外には何も到達できません。
- **ガードレールゲート。** すべての呼び出しは、実行前に `autonomy（自律レベル） × risk`
  で判定されます：

  | autonomy \ risk | read-only | mutating   |
  | --------------- | --------- | ---------- |
  | `read-only`     | 許可      | 禁止       |
  | `approval`      | 許可      | 承認が必要 |

- **読み取り専用のフロア。** 既定レベルは `read-only`。このレベルでは mutating な操作は
  ツールとして露出すらしません — エージェントは試せる書き込みを一切見られません。
- **書き込みは propose → approve。** `approval` レベルでも、mutating な操作は要求では
  実行されません。_提案として stage され_、別の承認ステップが実行します。モデルは
  一手で書き込みを行えない — 人間が構造的にループの中にいます。
- **定数コマンド・no-shell。** コマンドは固定の引数列として実行され（シェルを介さない）、
  モデルはコマンドの*どの部分も*供給しません — 引数すら。注入できる文字列が存在しません。
- **追記専用の監査ログ。** 何が提案され何が実行されたかを記録し、すべての操作を
  レビュー可能・巻き戻し可能に保ちます。

## 自律ラダー

ward はエージェントに許す範囲を一段ずつ広げます。上がるのは設定変更であって、
書き換えではありません：

1. **`read-only`** _(既定)_ — 観測のみ。基盤を変更できない。
2. **`approval`** — 変更を*提案*できる。各提案を人間が承認してから実行される。
3. **`autonomous`** — ポリシーの範囲内で、操作ごとの承認なしに動く。**未実装。**

## 現在地

初期段階で、進化の途中です。これまでの歩み：

- **M1** — MCP 経由の読み取り専用ステータス
- **M2** — 操作を YAML レジストリで宣言
- **M3** — 承認ゲートと初の mutating な操作
- **i18n** — ツールテキストとメッセージの二言語化（既定は英語、`WARD_LANG=ja`）

**インストール・クイックスタートのドキュメントは近日**、インターフェースが落ち着き
次第。それまでは [issues](https://github.com/ruribou/ward/issues) に計画が並んでいます。
