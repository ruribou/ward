# ward

> 自宅インフラを、自然言語で・安全に、Claude Code から操作する。

_English: [README.md](./README.md)_

**ward** は、AIエージェント（[Claude Code](https://claude.com/claude-code) 上の Claude）に、
自宅インフラの実際の操作 — ディスクの確認、コンテナの一覧、イメージの pull など — を
[MCP](https://modelcontextprotocol.io) ツールとして行わせる仕組みです。

難しいのは AI とマシンを繋ぐこと自体ではありません。**どこまで自律を許すか、そして
事故が起きたときにどう封じ込めるか** — ward はこの問いを中心に据えています。すべての
操作は、何かに触れる前に必ずガードレールを通る。自律は一段ずつ、意図して解放する。
まず読み取りだけ、次に人間の承認付き、完全な自律はいちばん最後です。

## 名前について — なぜ _ward_ か

この語は、プロジェクトの性格を二重に言い当てています。

- **ward（動詞）＝受け流す・防ぐ。** 事故を _ward off_ する。ガードレールで blast radius
  を封じ込める側面。
- **ward（名詞）＝後見下にある被後見人。** AI を最初は強い監督下に置き、信頼に応じて
  少しずつ自律を解放していく "ward" として扱う側面。

頭文字を拾えば **W**atched **A**utonomy, **R**eversible **D**eployment（監督付きの自律 ×
巻き戻せる変更）。プロジェクトの2本柱そのものです。

## 安全モデル

ward は既定で読み取り専用です。エージェントが行儀よくふるまう、という前提は置きません。
安全を担保するのは信頼ではなく、構造のほうです。

- **単一の真実源。** ward が実行できる操作は、すべて [`operations.yaml`](./operations.yaml)
  に固定のコマンドと `risk`（リスク）として宣言されています。この許可リストの外には、
  どんな操作も届きません。
- **ガードレールゲート。** すべての呼び出しは、実行前に `autonomy（自律レベル） × risk`
  で判定されます。

  | autonomy \ risk | read-only | mutating   |
  | --------------- | --------- | ---------- |
  | `read-only`     | 許可      | 禁止       |
  | `approval`      | 許可      | 承認が必要 |

- **読み取り専用のフロア。** 既定レベルは `read-only`。このレベルでは mutating な操作は
  ツールとして姿すら見せません。実行できない書き込みを、エージェントはそもそも目にしません。
- **書き込みは propose → approve。** `approval` レベルでも、mutating な操作は頼まれた
  だけでは動きません。いったん*提案として stage* され、別の承認ステップを踏んで初めて
  実行されます。モデルが一手で書き込むことはできず、人間が必ず間に入る構造です。
- **定数コマンド・no-shell。** コマンドは固定の引数列として実行し、シェルを介しません。
  しかもモデルはコマンドを一切組み立てません（引数ひとつ渡しません）。注入する隙が、
  そもそも存在しないということです。
- **追記専用の監査ログ。** 何を提案し、何を実行したかを記録します。すべての操作を、
  後から見直せて・巻き戻せる状態に保ちます。

## 自律ラダー

ward は、エージェントに許す範囲を一段ずつ広げていきます。段を上げるのは設定の変更で
あって、作り直しではありません。

1. **`read-only`** _(既定)_ — 見るだけ。基盤には手を出せない。
2. **`approval`** — 変更を*提案*できる。提案はひとつずつ人間が承認してから実行される。
3. **`autonomous`** — ポリシーの範囲内なら、操作ごとの承認なしで動く。**未実装。**

## 現在地

まだ初期段階で、育てている最中です。ここまでの歩み:

- **M1** — MCP 経由の読み取り専用ステータス
- **M2** — 操作を YAML レジストリで宣言
- **M3** — 承認ゲートと、初の mutating な操作
- **i18n** — ツールの文言とメッセージの二言語化（既定は英語、`WARD_LANG=ja` で日本語）

**インストールとクイックスタートのドキュメントは、これから** — インターフェースが
落ち着き次第そろえます。それまでの計画は [issues](https://github.com/ruribou/ward/issues)
にあります。
